## 四、服务器故障处理

### 1. OS 无限重启（rc.local 循环 reboot）
- **背景**：旧系统（nvme5n1）`/etc/rc.local` 里有 reboot 500 cycles 脚本，GRUB 有密码保护无法进单用户模式
- **方案**：在第二块盘（nvme0n1）临时装新系统作为救援盘
- **操作步骤**：
  ```bash
  vgchange -ay openeuler
  mount /dev/openeuler/root /mnt
  sed -i 's/reboot/#reboot/g' /mnt/etc/rc.local
  chmod -x /mnt/etc/rc.local
  umount /mnt
  ```
- 重启后 BIOS 切回旧盘正常启动，不再循环
- 清理临时系统盘：`dd if=/dev/zero of=/dev/nvme0n1 bs=1M count=10`
- **经验**：BIOS 里改网卡 IP 对 OS 无效（那是给 PXE 用的）

### 2. 静态 IP 配置
- 当前 DHCP IP：`10.121.177.157/23`，网关 `10.121.176.1`
- 发现实际管理连接是 `endvnic` 设备，不是 `eno1`，DNS：`10.10.10.113, 10.16.10.6`
- 固定 IP 命令：
  ```bash
  nmcli con modify endvnic ipv4.addresses 10.121.177.157/23
  nmcli con modify endvnic ipv4.gateway 10.121.176.1
  nmcli con modify endvnic ipv4.dns "10.10.10.113,10.16.10.6"
  nmcli con modify endvnic ipv4.method manual
  nmcli con up endvnic
  ```

### 3. 25G 光口 NO-CARRIER 排查
- **现象**：eno5/eno6 状态 `NO-CARRIER`，但对端已连接
- **原因**：`ethtool` 显示 `Link detected: no (Logical mismatch, RS FEC is not locked)`
- **根因**：两端 25G 光口 FEC（前向纠错）模式不匹配
- **解决方向**：`ethtool --set-fec eno5 encoding rs` 或 `baser`，两端保持一致

### 4. Codex CLI 无法启动
- **现象**：`codex` 报错 `CODEX_HOME points to "... but that path does not exist`
- **原因**：环境变量 `CODEX_HOME` 指向旧路径 `C:\Users\...`，项目已搬到 `E:\桌面\...`
- **解决**：更新环境变量 + `config.toml` 项目信任路径

### 5. RAID 热备盘（Hot Spare）知识
- 全局热备 vs 专用热备
- 硬性条件：容量 ≥ 阵列最小盘、接口类型一致、介质类型匹配
- 不强制同品牌，但建议同型号（性能一致性）
