## 一、FRU 信息修改问题排查

### 1. M2 转接板 FRU 写入无效果
- **现象**：`ipmitool fru edit 4 field b 0 iSoftStone123` 执行成功，但 `ipmitool fru print 26`（M2TransferCard5）信息未变
- **原因**：编辑的是 ID 4（PSUBoard2），检查的是 ID 26（M2 转接板），不是同一设备
- **正确操作**：应使用 `ipmitool fru edit 26 field b 0 "iSoftStone123"`

### 2. ipmitool 段错误（Segmentation fault）
- **现象**：重复执行 `fru edit 4` 时出现 `Segmentation fault（核心已转储）`
- **原因**：ipmitool 连续写同一 FRU 时缓冲区状态异常或 OEM 版本边界处理 bug
- **建议**：每次写入后间隔几秒，避免连续快速写入；确认设备 ID 正确再操作

### 3. ipmitool FRU 字段说明
| 字段 | 含义 |
|------|------|
| `field b` | Board 区域（Board Mfg、Board Product 等） |
| `field p` | Product 区域（Product Name、Serial、Part Number 等） |
| `field c` | Chassis 区域 |

## 二、PSU FRU 信息批量查询

### 命令示例
```bash
# 列出所有 PSU 设备 ID
ipmitool fru list | grep -i psu

# 批量打印所有 PSU 详细信息
for id in 3 4 13 14 12 15 16 17; do
  echo "=== PSU ID $id ==="
  ipmitool fru print $id
  echo ""
done

# 只筛选关键字段
for id in 3 4 13 14 12 15 16 17; do
  ipmitool fru print $id
done | grep -i "mfg\|serial\|product\|part\|type"
```
