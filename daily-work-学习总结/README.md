# 日常沉淀区说明

## 这个目录放什么

这里用于保存每天的复盘、学习稿和现场案例。

和 `docs/` 的区别是：

- `daily-work-学习总结/` 更偏当天输入和阶段性沉淀
- `docs/` 更偏长期稳定的知识库

## 当前内容类型

### 1. 每日复盘

命名建议：

- `YYYY-MM-DD-工作复盘.md`

适合记录：

- 当天问题
- 分析过程
- 结论
- 后续动作

### 2. 学习稿 / Day 文档

命名建议：

- `Day13-xxx.md`
- `Day14-xxx.md`

适合记录：

- 当天学习主题
- 方法论提炼
- 面试式自测题

### 3. 案例稿

命名建议：

- `案例-xxx.md`

适合记录：

- 真实现场故障
- 定位链路
- 闭环结果
- 可沉淀到手册的点

### 4. 原始工作日志

目录：

[logs-每天工作日志](E:/72785/Codex/SIT提升/Server-SIT-Mastery/daily-work-学习总结/logs-每天工作日志)

适合放：

- 原始命令输出
- 现场日志摘录
- 不适合直接放进正式手册的原始记录

## 使用建议

如果一份内容经过整理后已经具有通用性，就继续往 `docs/manual/` 或 `docs/100-day-plan/` 提升，不要一直停留在日常区。
