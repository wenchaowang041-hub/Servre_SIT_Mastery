# 2026-04-16 工作记录

## 一、NVMe 热插拔 FIO 时序 Bug 修复（plug-unplug-safe）

### 1. 发现 FIO 时序错误
- **现象**：跑了 9 圈热插拔测试，第 10 圈才发现 FIO 只在第 1 圈跑了 60 秒就退出
- **根因**：`auto-plug-unplug-fio-safe.sh` 原流程中 FIO 在循环外启动一次，runtime=60s，循环内没有任何 FIO 压力
- **修复**：FIO 改为每轮循环内启动，覆盖该轮所有盘的拔插时间，结束后 `pkill` 清理残留

### 2. FIO 参数对齐操作手册
- `runtime` 从 60 改为 300（默认值，可被 `FIO_RUNTIME` 覆盖）
- 新增 `--eta=never` 参数
- 修复 `fio.sh` 中 nvme8n12 拼写错误（原为 `nvme8n12`，缺 p2 后缀）
- 改为 `FIO_RUNTIME` 环境变量动态控制

## 二、测试前/后日志采集增强

### 1. 预采集脚本（2-check-start-safe.sh）

### 2. 后采集脚本（5-check-log-safe.sh）

## 三、插盘后改为轮询检测（wait_for_disk）

- 旧版：固定等 6 秒，不管盘有没有被系统识别
- 新版：`wait_for_disk()` 轮询检测 `/dev/nvmeXn1 + p1 + p2` 是否存在
- 超时 30 秒（默认），超时后告警但不阻塞流程
- 好处：识别快的盘少等时间，识别慢的盘至少能发现问题

## 四、改为批量拔插模式

- 旧流程：逐盘拔插（每块盘单独 拔→等→插→MD5）
- 新流程：FIO → 拔所有盘 → 等 → 插回所有盘 → 逐个检测识别 → MD5（一次校验所有盘）
- MD5 校验从每循环 11 次（逐盘调用）改为每循环 1 次（内部遍历所有盘）

## 五、清理脚本（9-cleanup-safe.sh）

- 一键清理热插拔测试环境：卸载挂载 → 备份+清理 fstab → sgdisk+wipefs+dd 三重清分区 → 打印 lsblk 确认
- 系统盘 nvme5 自动跳过
- 去掉了无用的 rescan 步骤（`/sys/class/nvme/nvme*/rescan` 只读不可写）

## 六、Notify 通知式热插拔脚本重写

### 1. 整体重写，对齐 batch-safe 逻辑
- 主流程：分区 → UUID → 预采集日志 → MD5源文件 → 循环 → 最终采集
- 所有子脚本排除系统盘 nvme5（原 `for i in {0..11}` 包含系统盘）

### 2. slot 映射调试
- 通过 `readlink /sys/block/nvme0n1/device` 尝试匹配 PCI 插槽，发现返回 `../../nvme0`
- 改为 `/sys/block/nvmeXn1/device/nvme0` 的完整路径提取控制器 BDF
- 正确路径：`readlink -f` → `/sys/devices/pci0000:06/0000:06:04.0/0000:09:00.0/nvme/nvme0`
- 取倒数第二个目录名 `0000:09:00.0` → 去 function → `0000:09:00` → 和 slot address 精确比对

### 3. 单盘 slot 断电验证
- nvme0 对应 slot 0-2（address: 0000:09:00）
- `echo 0 > /sys/bus/pci/slots/0-2/power` → 盘从 lsblk 消失
- `echo 1 > /sys/bus/pci/slots/0-2/power` → 盘恢复（分区结构完整）
- 验证成功：软件 PCI 断电有效

## 七、dmesg AER 错误分析

### 带IO热插拔测试中 dmesg 发现
- `07:00.0` 和 `ca:00.0` 两块盘反复出现 `device recovery failed`
- 多个 Root Port（06:00.0, 06:02.0, c7:04.0, e3:00.0）报告 Uncorrected (Non-Fatal) 错误
- AC 重启后所有盘恢复正常，故障灯熄灭
- **判断**：这些是热插拔过程中的 PCIe 链路瞬断导致的 AER 错误，非硬件损坏
