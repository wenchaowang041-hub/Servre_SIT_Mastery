# Server-SIT-Mastery

## 项目定位

这个仓库用于持续沉淀服务器硬件整合测试（SIT）学习、现场问题、方法论手册和百日计划资料。

当前主线目标：

- 6个月内提升到服务器硬件整合测试高级工程师水平
- 主攻方向：鲲鹏 / 昇腾 / AI 服务器 / 整机 SIT

## 目录分工

### 1. `docs/manual/`

主知识库。

适合放：

- 整机 SIT 方法论
- CPU / 内存 / 网络 / 存储 / BMC / NPU 等专题手册
- 项目 bring-up 流程
- 标准检查清单

### 2. `docs/100-day-plan/`

百日计划资料库。

适合放：

- 训练计划表
- Day 原始文档
- 已发表博客
- 复习提纲
- 配套手册资料

### 3. `daily-work-学习总结/`

日常沉淀区。

适合放：

- 每日复盘
- 在家补习稿
- 当天学习稿
- 现场案例

### 4. `practice/`

练手脚本、自动化脚本、临时实验代码。

### 5. `interview/`

面试题、问答沉淀、标准回答。

## 推荐使用规则

### 规则1：聊天里形成的内容，尽量落文件

不要长期只放在对话里。

### 规则2：先分“手册 / 计划 / 复盘 / 案例”

这样不会把知识库和当天现场内容混在一起。

### 规则3：先提交到 Git，再切电脑

保证公司电脑和家里电脑都能续上。

## 一句话原则

线程可以换，仓库不能乱；记忆会模糊，Git 里的沉淀不能断。
