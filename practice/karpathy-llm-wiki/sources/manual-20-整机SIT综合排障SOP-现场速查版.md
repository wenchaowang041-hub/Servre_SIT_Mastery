# 整机 SIT 综合排障 SOP：现场速查版

## 1. 使用目的

这份文档不是完整教学版，而是现场快速判断时用的一页式速查版。

适合场景：

- 新项目 bring-up
- 设备不识别
- 性能不达标
- 机器状态异常
- 压测前快速判断是否具备继续测试条件

## 2. 总原则

先分类型，再分层，再找证据，最后下结论。

不要一上来就说：

- 硬件坏了
- 系统有问题
- 驱动不行

## 3. 先分三类问题

### A. 设备识别异常

典型现象：

- `lspci` 看不到
- 工具认不到
- 设备部分存在、部分缺失

### B. 性能异常

典型现象：

- 吞吐低
- 延迟高
- 抖动大
- 带宽掉

### C. 机器状态异常

典型现象：

- 过温
- 风扇异常
- 掉电
- 重启
- 告警多

## 4. 再分五层

```text
硬件
-> BIOS / BMC / FW / CPLD
-> Driver
-> OS
-> Tool
```

现场先问自己：

1. 是工具问题吗
2. 是 OS / kernel 问题吗
3. 是驱动问题吗
4. 是固件 / 版本问题吗
5. 最后才是硬件本体问题吗

## 5. 现场第一轮必查命令

### 识别问题先查

```bash
lspci
lsmod
dmesg
journalctl -k
```

### 性能问题先查

```bash
ethtool <port>
lspci -vv
lscpu
numactl -H
cat /proc/interrupts
top
```

### 健康问题先查

```bash
ipmitool sensor
ipmitool sel list
dmesg
journalctl -k
```

## 6. 三类问题的速查判断

### A. 设备识别异常速查

#### 场景1

- `lspci` 能看到
- `lsmod` 没驱动

优先判断：

- 驱动未安装
- 驱动未加载
- 驱动和 kernel 不匹配

#### 场景2

- `lspci` 看不到

优先判断：

- 插槽
- 接线
- 供电
- 板卡
- FW / CPLD / BIOS / BMC
- 平台适配关系

#### 补查项

- 槽位是否插对
- 接线拓扑是否正确
- 板间通信是否正常
- 版本矩阵是否对齐

### B. 性能异常速查

#### 先问

- 是链路问题，还是处理路径问题

#### 链路层先看

- 网口是否 `UP`
- 速率是否正确
- 双工是否正常
- 光模块 / 网线 / 交换机是否异常
- PCIe 是否降速降宽

#### 路径层再看

- 设备挂在哪个 NUMA
- IRQ 落在哪些 CPU
- 业务线程跑在哪些 CPU
- 是否跨 NUMA / 跨 socket
- 是否有 CPU 热点
- 队列是否过深

#### 核心提醒

```text
网口 UP，不等于性能健康
```

### C. 健康异常速查

#### 先看

- BMC Web
- `ipmitool sensor`
- `ipmitool sel list`

#### 判断逻辑

- `sensor` 看现在
- `SEL` 看过去

#### 如果当前正常但历史连续异常

例如：

- `Temp high`
- `PSU lost`
- `Fan fault`

处理原则：

- 先停烧机
- 先停高压
- 先查散热、风道、风扇、功耗、环境温度

## 7. 版本矩阵速查

现场至少核对：

- BIOS
- BMC
- Device FW
- CPLD
- Driver
- Kernel
- OS

不要单看一个版本是否正常，要看组合是否匹配。

## 8. 现场操作前必须记住

在做下面动作前，先保留原始证据：

- 升级 BIOS / BMC / FW
- 换驱动
- 换线
- 换卡
- 换槽位
- 改 BIOS 设置

建议先保留：

```bash
dmesg
journalctl -k
ipmitool sensor
ipmitool sel list
lspci -vv
lsblk
ethtool
```

## 9. 现场最短排障话术

你可以直接用这套说法：

我会先把问题分成识别异常、性能异常和健康状态异常三类，再判断它属于硬件、固件、驱动、OS 还是工具层，然后用日志、BMC、拓扑和版本信息去找证据，最后再确认是配置、版本、路径还是硬件本体故障。

## 10. 最短记忆版

只记这四句：

1. 先分识别、性能、健康三类问题。
2. 再分硬件、固件、驱动、OS、工具五层。
3. 先拿证据，不先拍脑袋。
4. 最后再下结论，不把临时判断当最终结论。
