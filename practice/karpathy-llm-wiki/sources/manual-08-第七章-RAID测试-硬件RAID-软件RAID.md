# 第七章 RAID 测试（硬件 RAID、软件 RAID）

## 1. 模块概述与重要性

RAID 测试不是“阵列建出来就结束”。  
真正需要验证的是：

- 功能是否正确
- 冗余是否生效
- 性能是否可接受
- 坏盘后能否恢复
- 重建期间业务影响有多大

高级工程师做 RAID 测试，不只是看能不能创建 RAID 0/1/5/10，而是会问：

- 写回缓存是否安全
- 重建速度是否影响业务
- 单盘故障时阵列状态是否正确
- 控制器缓存、条带大小、盘型是否合理

## 3. 基础原理讲解

### 3.1 RAID 测试到底要看什么

1. 阵列创建是否成功
2. 阵列状态是否健康
3. 正常态性能如何
4. 降级态性能如何
5. 重建态性能如何
6. 坏盘、换盘、回迁是否正确

### 3.2 为什么 RAID 测试必须区分三种状态

很多人只测正常态，但真正的生产环境风险在：

- 降级态
- 重建态

因为这时性能会下降、时延会增大、告警会增加，最容易暴露控制器、盘和缓存策略问题。

## 4. 详细测试用例

### 用例 1：阵列创建与状态检查

#### 所需工具

- `storcli`
- `mdadm`
- `cat /proc/mdstat`

#### 完整测试命令

```bash
storcli show 2>/dev/null || true
storcli /c0 /vall show 2>/dev/null || true
cat /proc/mdstat
mdadm --detail --scan 2>/dev/null || true
```

#### 重点输出字段表

| 字段 | 含义 | 正常范围 | 异常处理 |
|---|---|---|---|
| `State` | 阵列状态 | Optimal/Active | Degraded：查坏盘和重建 |
| `VD` / `Array` | 虚拟盘/阵列标识 | 应完整可见 | 缺失：查控制器或配置 |

### 用例 2：正常态性能测试

```bash
fio --name=raidtest \
    --filename=/mnt/raid/fio.test \
    --size=20G \
    --rw=randrw \
    --rwmixread=70 \
    --bs=4k \
    --iodepth=64 \
    --ioengine=libaio \
    --direct=1 \
    --runtime=120 \
    --time_based \
    --group_reporting
```

### 用例 3：降级态与重建态观察

#### 测试步骤

1. 记录正常态性能。
2. 模拟单盘故障或离线。
3. 观察阵列进入 Degraded。
4. 更换盘并触发 Rebuild。
5. 记录重建速度和性能衰减。

#### 完整测试命令

```bash
cat /proc/mdstat
mdadm --detail /dev/md0 2>/dev/null || true
storcli /c0 /vall show all 2>/dev/null || true
```

## 5. 结果分析与问题诊断方法

先看控制器状态，再看盘状态，再看缓存策略，再看重建速度和业务影响。

## 6. 最佳实践与注意事项

1. RAID 测试必须报告正常态、降级态、重建态。
2. 写回缓存测试前先确认断电保护。
3. 条带大小和业务模型必须匹配。
