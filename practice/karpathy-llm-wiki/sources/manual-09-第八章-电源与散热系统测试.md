# 第八章 电源与散热系统测试

## 1. 模块概述与重要性

电源和散热系统决定了一台服务器能不能长期稳定输出设计性能。  
这件事在 AI 服务器、GPU/NPU 服务器和高核数 CPU 平台上尤其重要。

很多现场问题表面上看像：

- CPU 跑分低
- GPU/NPU 算力下降
- 网卡吞吐波动
- 系统偶发重启

实际上根因可能是：

- PSU 冗余异常
- 输入功率受限
- 风扇策略过保守
- 机房温度过高
- 冷板或液冷流量异常
- BMC 传感器或控制策略不合理

所以这一章不是附属章，而是整机稳定性判断的核心章。

## 3. 基础原理讲解

### 3.1 为什么功耗和热是整机级问题

因为电源和散热会同时影响：

- CPU 频率
- GPU/NPU 功耗墙
- 网卡和 SSD 的热状态
- 风扇噪音和系统可靠性

### 3.2 典型三类风险

1. 供电风险
   - PSU 不冗余、输入异常、功率边界不足
2. 温度风险
   - 环境温度高、风道差、散热器接触不良
3. 控制策略风险
   - BMC 风扇曲线偏保守或偏激进

## 4. 详细测试用例

### 用例 1：PSU、风扇、温度传感器基线检查

```bash
ipmitool sensor
ipmitool sdr list full
ipmitool sel list
```

| 字段 | 含义 | 正常范围 | 异常处理 |
|---|---|---|---|
| `PSU` 状态 | 电源状态 | 正常、冗余齐全 | 异常：检查供电和模块 |
| `Fan` | 风扇状态/转速 | 应在合理范围 | 偏低/缺失：查风扇和策略 |
| `Temp` | 温度 | 不应逼近阈值 | 高温：查风道和环境 |

### 用例 2：满载功耗与温度观察

```bash
stress-ng --cpu 0 --timeout 30m --metrics-brief
ipmitool sensor
```

### 用例 3：PSU 冗余切换测试

#### 测试步骤

1. 记录双 PSU 正常状态。
2. 人工断开一路电源。
3. 观察系统是否继续运行。
4. 检查传感器和 SEL 是否记录正确。
5. 恢复供电后确认冗余恢复。

### 用例 4：风扇策略与热降频联动检查

```bash
ipmitool sensor
dmesg -T | egrep -i "thermal|throttle"
```

## 5. 结果分析与问题诊断方法

先看传感器，再看 SEL，再对照性能衰减时间线。  
如果前期性能正常、后期持续下降，优先怀疑热边界或功耗墙。

## 6. 最佳实践与注意事项

1. 功耗和散热测试必须与压力测试一起做。
2. 传感器异常时，不要直接做性能结论。
3. AI 服务器一定保留温度、功耗、频率时间序列。
