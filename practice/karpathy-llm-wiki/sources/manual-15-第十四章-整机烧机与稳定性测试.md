# 第十四章 整机烧机与稳定性测试（长时间压力测试、混合负载）

## 1. 模块概述与重要性

整机烧机是 SIT 的收敛点。  
前面各模块单独通过，不代表整机在真实边界下也能稳定通过。

整机烧机真正要验证的是：

- 高负载下会不会降频
- 长时间运行会不会报错
- CPU/内存/网络/存储/GPU/NPU 是否会互相放大问题
- 功耗和散热是否足以支撑长稳

## 2. 2026年厂商与主流产品信息对比表

| 平台方向 | 典型平台 | 典型特点 | 重点测试关注点 |
|---|---|---|---|
| 通算服务器 | Kunpeng/Xeon/EPYC 双路平台 | 高核数、大内存 | CPU+内存+盘+网混合长稳 |
| AI 服务器 | Ascend/GPU 平台 | 高功耗、高热密度 | 加速器+网络+散热+供数 |

## 3. 基础原理讲解

烧机测试核心不是“跑多久”，而是“是否在时间维度暴露边界问题”。

最常见边界问题：

- 热降频
- PSU/风扇告警
- ECC 计数增长
- 网卡重传和错误增长
- NVMe reset
- GPU/NPU 掉卡

## 4. 详细测试用例

### 用例 1：CPU+内存混合烧机

```bash
stress-ng --cpu 0 --vm 8 --vm-bytes 80% --timeout 8h --metrics-brief
```

### 用例 2：存储+网络混合负载

```bash
fio --name=soak --filename=/data/fio.test --size=50G --rw=randrw --bs=4k --iodepth=64 --runtime=8h --time_based --group_reporting
iperf3 -c <server_ip> -P 8 -t 28800
```

### 用例 3：AI 平台整机长稳

步骤：
1. GPU/NPU 满载
2. Host CPU 保持高负载
3. 网络持续通信
4. 每 5 分钟记录一次温度、功耗、频率、错误

## 5. 结果分析与问题诊断方法

重点看时间序列：

- 前期正常、后期下降：偏热或功耗
- 随机掉卡：偏 PCIe/固件/供电
- 错误计数增长：偏长期稳定性边界

## 6. 最佳实践与注意事项

1. 烧机必须采时间序列，不要只看工具退出状态。
2. 压力组合要接近真实业务。
3. 烧机结果必须与传感器和日志关联分析。
