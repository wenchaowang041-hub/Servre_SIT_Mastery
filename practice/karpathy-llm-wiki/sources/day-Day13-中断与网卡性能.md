# Day13：中断与网卡性能

## 对齐计划表

- 阶段：Phase1-网络
- 周次：W2
- 模块：IRQ
- 主题：中断与网卡性能

## 今日目标

- 理解 IRQ、网卡中断、软中断之间的关系
- 建立 `IRQ -> CPU -> NUMA -> 网卡性能` 的基础概念
- 学会从 `/proc/interrupts` 里定位网卡中断落在哪些 CPU

## 先记住一句话

网卡性能不只是看网口速率和链路状态，还要看中断被打到哪些 CPU 上，以及这些 CPU 和网卡所在 NUMA 是否匹配。

## 一、什么是 IRQ

IRQ 可以先简单理解成“设备向 CPU 发出的中断请求”。

对网卡来说，收到包、发包完成、队列事件等，都可能触发中断。  
CPU 收到中断后，才会继续处理后续的数据路径。

所以网卡性能并不是只有：

- 网卡型号
- 链路速率
- 驱动版本

还和下面这些有关：

- 中断落在哪些 CPU
- CPU 是否忙
- CPU 是否和设备 NUMA 接近
- 中断是否分布均衡

## 二、为什么 IRQ 会影响网卡性能

如果中断落点不合理，常见后果包括：

- 某几个 CPU 被打满
- 吞吐跑不上去
- 抖动变大
- 延迟变差
- 跨 NUMA 访问变多

你要建立这个基本概念：

设备在 NUMA1，结果中断主要落在 NUMA0 的 CPU 上，这通常就不是最优路径。

## 三、今天主要看什么

今天不追求把所有网络调优做完，只先做三件事：

1. 找到网卡相关中断
2. 看这些中断落到哪些 CPU
3. 初步判断这些 CPU 是否靠近网卡所在 NUMA

## 四、核心命令

```bash
cat /proc/interrupts
grep eno /proc/interrupts
grep -i eth /proc/interrupts
grep -i mlx /proc/interrupts
```

补充建议：

```bash
lscpu
numactl -H
ethtool -i <port>
ethtool -S <port>
```

## 五、怎么读 `/proc/interrupts`

`/proc/interrupts` 里每一列通常对应一个 CPU，行里会显示：

- 中断号
- 每个 CPU 收到的中断次数
- 中断控制器类型
- 设备或驱动名称

你重点不是把整张表背下来，而是看：

- 哪些行属于网卡
- 哪些 CPU 的计数在增长
- 是否只集中在少数几个 CPU

## 六、今天的标准训练顺序

### 第1步：确认网口和驱动

先确认你观察的是哪个网口，不要看错设备。

例如：

```bash
ip link
ethtool -i eno1
```

重点记住：

- 网口名
- 驱动名
- bus-info

这能帮助你后面把网口、PCIe 地址和中断对应起来。

### 第2步：从 `/proc/interrupts` 里找网卡中断

你可以先粗搜：

```bash
cat /proc/interrupts
```

再做针对性过滤：

```bash
grep eno /proc/interrupts
grep -i mlx /proc/interrupts
grep -i eth /proc/interrupts
```

不同平台显示名称可能不一样，所以不要死盯某一个关键字。

### 第3步：记录中断落在哪些 CPU

你要记录的不是“有中断”，而是：

- 中断主要落在哪几个 CPU
- 是不是分布很偏
- 是否只压在少量核上

如果某个网卡队列的中断明显集中在极少数 CPU 上，就要提高警惕。

### 第4步：和 NUMA 做一次对照

这一步是今天最关键的升级点。

结合：

```bash
lscpu
numactl -H
```

你要回答：

- 这些 CPU 属于哪个 NUMA 节点
- 网卡大概率挂在哪个 NUMA
- 中断落点和设备 NUMA 是否接近

今天先形成概念，不一定一次就做到完全精确。

## 八、今天最容易犯的错

### 错误1：只看链路，不看中断

链路是 `UP`，不代表性能路径已经合理。

### 错误2：只看有没有中断，不看落在哪些 CPU

真正影响性能的，往往是分布和亲和性，不只是“有没有”。

### 错误3：只看 CPU 忙不忙，不看 NUMA

CPU 没打满，不代表路径就优。跨 NUMA 一样可能拖慢性能。

## 十、面试式自测题

### 题1

为什么网卡中断会影响网络吞吐？

标准答案方向：

- 因为数据收发需要 CPU 处理中断和后续协议栈路径
- 中断落点不合理会导致 CPU 热点、抖动和跨 NUMA 访问

### 题2

看到网口 `UP` 之后，为什么还不能直接认为网络路径没问题？

标准答案方向：

- `UP` 只能说明链路建立
- 不代表中断分布合理
- 不代表性能一定达标

### 题3

今天你最该建立的关系是什么？

标准答案方向：

- `IRQ -> CPU -> NUMA -> 网卡性能`
