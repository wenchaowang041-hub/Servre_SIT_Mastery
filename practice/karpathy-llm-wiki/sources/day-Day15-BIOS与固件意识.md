# Day15：BIOS 与固件意识

## 对齐计划表

- 阶段：Phase1-系统
- 周次：W3
- 模块：BIOS/固件
- 主题：BIOS 与固件意识

## 今日目标

- 理解 BIOS、BMC、Firmware、Driver、OS 分层
- 建立“不是所有问题都叫硬件问题”的意识
- 知道 BIOS 会影响哪些关键系统行为

## 先记住一句话

很多服务器问题，不是坏硬件，也不是坏系统，而是“版本、配置、分层关系没对齐”。

## 一、为什么 Day15 很关键

你前面已经学了：

- Day13：网络性能路径
- Day14：BMC 与健康状态

那 Day15 往前再补一层，解决的是：

**为什么同一台机器，换个 BIOS 设置、换个固件版本，行为就会不一样。**

这也是整机 SIT 和普通运维视角的一个区别。

## 二、先把五层分清楚

你可以先把服务器分成这几层：

```text
硬件
-> BIOS / BMC / 设备固件
-> Driver
-> OS
-> 工具 / 应用
```

### 1. 硬件层

包括：

- CPU
- 内存
- NVMe
- 网卡
- NPU
- 主板
- PSU

### 2. 固件层

包括：

- BIOS
- BMC
- NVMe 固件
- 网卡固件
- RAID 卡固件
- CPLD 等板级版本

### 3. 驱动层

包括：

- 网卡驱动
- NVMe 驱动
- NPU 驱动
- 其他内核模块

### 4. OS 层

包括：

- Linux 内核
- systemd / NetworkManager
- 块设备、网口、进程、调度

### 5. 工具层

包括：

- `lscpu`
- `ethtool`
- `ipmitool`
- `fio`
- `iperf3`
- `npu-smi`

## 三、为什么要建立分层意识

因为同一个现象，可能来自不同层。

例如：

- CPU 数量不对
  - 可能是 BIOS 配置
  - 也可能是 OS 没识别完整
- 网卡性能差
  - 可能是驱动
  - 也可能是 BIOS 里的 PCIe 相关配置
- NPU 不识别
  - 可能是 PCIe 层
  - 也可能是驱动、固件、工具链层

所以 Day15 最重要的不是命令，而是：

**先判断问题可能属于哪一层。**

## 四、BIOS 典型会影响什么

你现在最该记住的是 BIOS 不是“能进设置界面就结束了”，它会直接影响下面这些东西：

- CPU 线程开关
- NUMA 划分
- 启动模式
- PCIe 相关行为
- 功耗策略
- 内存策略
- 虚拟化/IOMMU 相关设置

### 常见例子

#### 1. CPU 线程数变化

如果 SMT/超线程开关变化，`lscpu` 看到的逻辑 CPU 数就会变。

#### 2. NUMA 行为变化

不同 BIOS 配置可能影响 NUMA 暴露方式或分区方式。

#### 3. PCIe 行为变化

某些平台 BIOS 设置会影响：

- 链路速率
- bifurcation
- slot 行为
- 设备枚举稳定性

#### 4. 功耗和频率行为变化

功耗策略不同，可能直接改变：

- 峰值性能
- 稳定性
- 温度表现

## 五、今天建议建立的判断方法

看到一个问题，不要直接说“硬件坏了”，先问：

1. 这是硬件层问题吗
2. 会不会是 BIOS / BMC / 固件层问题
3. 会不会是驱动层问题
4. 会不会只是 OS / 工具层看到的结果

这就是 Day15 的核心。

## 六、今天建议记住的 BIOS 检查点

如果你后面接触真实服务器，遇到异常时，BIOS 至少要关注这些检查项：

- BIOS 版本
- 启动模式
- CPU 线程配置
- NUMA / SNC / NPS 类配置
- PCIe 相关配置
- 功耗模式
- 虚拟化 / IOMMU

## 七、和你当前工作怎么连接

你现在做的新项目 bring-up、PCIe 设备识别、网卡槽位适配、NPU 安装，实际上已经在碰固件层问题了。

Day15 要你建立的意识就是：

**以后不要只说“设备不识别”，要开始说清楚是 BIOS、BMC、固件、驱动还是 OS 哪一层出了问题。**

## 八、面试式自测题

### 题1

为什么服务器问题不能都归类成“硬件问题”？

标准答案方向：

- 因为现象可能来自 BIOS、固件、驱动、OS 等不同层

### 题2

BIOS 会影响哪些你平时能直接观察到的结果？

标准答案方向：

- CPU 线程数
- NUMA 结构
- PCIe 行为
- 功耗和频率表现

### 题3

Day15 最重要的能力是什么？

标准答案方向：

- 分层判断问题来源

## 九、今天的补习结论

- Day15 的重点不是多学几个 BIOS 名词，而是建立分层意识。
- 你要从“看到现象就下结论”升级到“先判断属于哪一层”。
- 这一步一旦建立起来，后面做整机 SIT、NPU 排障、网络和存储定位都会稳很多。
