## 2. 当前机器已确认信息


### 2.2 当前 NVMe 拓扑分组

根据你现场执行的 `lspci -tv` 和 `/sys/class/nvme` 映射，当前 12 块 NVMe 分为 3 组拓扑。

#### 第一组拓扑：`RootPort = pci0000:06`

```bash
/dev/nvme0n1 -> NVMe:0000:07:00.0
/dev/nvme1n1 -> NVMe:0000:08:00.0
/dev/nvme2n1 -> NVMe:0000:09:00.0
/dev/nvme3n1 -> NVMe:0000:0a:00.0
```

#### 第二组拓扑：`RootPort = pci0000:c7`

```bash
/dev/nvme4n1 -> NVMe:0000:c8:00.0
/dev/nvme5n1 -> NVMe:0000:c9:00.0   # 系统盘，不能动
/dev/nvme6n1 -> NVMe:0000:ca:00.0
/dev/nvme7n1 -> NVMe:0000:cb:00.0
```

#### 第三组拓扑：`RootPort = pci0000:e3`

```bash
/dev/nvme8n1  -> NVMe:0000:e4:00.0
/dev/nvme9n1  -> NVMe:0000:e5:00.0
/dev/nvme10n1 -> NVMe:0000:e6:00.0
/dev/nvme11n1 -> NVMe:0000:e7:00.0
```

## 3. 这项测试到底怎么覆盖

“被测硬盘遍历所有拓扑不同的槽位”的意思不是把两块测试盘遍历到所有非系统盘槽位。

正确理解是：

- 覆盖所有不同 PCIe 拓扑组
- 每种拓扑至少做一整轮完整测试
- 每轮使用 2 块被测盘

对你这台机器，最实用的覆盖方案是 3 轮：

### 第一轮：覆盖 `pci0000:06`

建议被测盘：

```bash
/dev/nvme0n1
/dev/nvme1n1
```

### 第二轮：覆盖 `pci0000:e3`

建议被测盘：

```bash
/dev/nvme8n1
/dev/nvme9n1
```

### 第三轮：覆盖 `pci0000:c7`

建议被测盘：

```bash
/dev/nvme4n1
/dev/nvme6n1
```

注意：

- `/dev/nvme5n1` 是系统盘，不能作为被测盘
- `nvme4n1 + nvme6n1` 只是建议组合，`nvme4n1 + nvme7n1` 也可以

## 7. 每轮测试的标准执行顺序

下面是现场最实用的简化版。

### 7.1 首次针对本轮两块盘执行一次初始化

进入本轮日志目录，例如第一轮：

```bash
cd /root/hotplug_logs/2026-04-09_topo06
chmod +x /root/nvme_hotplug_scripts/*.sh
/root/nvme_hotplug_scripts/0-prepare-nvme.sh
```

说明：

- `0-prepare-nvme.sh` 会清空 `DUTS` 中这两块盘的数据
- 会重新分区：
  - `p1 = 10GB`
  - `p2 = 剩余全部`
- `p1` 会格式化成 `ext4`

### 7.2 每轮正式开始

```bash
cd /root/hotplug_logs/2026-04-09_topo06
/root/nvme_hotplug_scripts/1-check-Begin.sh
/root/nvme_hotplug_scripts/2-md5.sh
/root/nvme_hotplug_scripts/fio.sh
```

### 7.3 然后人工交替拔插

第一块盘：

1. 拔出第 1 块被测盘
2. 等 30 秒
3. 慢速插回
4. 再等 6 秒
5. 执行：

```bash
cd /root/hotplug_logs/2026-04-09_topo06
/root/nvme_hotplug_scripts/3-check-md5.sh
```

第二块盘：

1. 拔出第 2 块被测盘
2. 等 30 秒
3. 慢速插回
4. 再等 6 秒
5. 执行：

```bash
cd /root/hotplug_logs/2026-04-09_topo06
/root/nvme_hotplug_scripts/3-check-md5.sh
```

### 7.4 重复到 30 次

后续按下面节奏一直重复：

1. 拔盘
2. 等 30 秒
3. 慢插回去
4. 等 6 秒
5. 执行 `3-check-md5.sh`

直到总次数达到 30 次。

### 7.5 本轮结束

```bash
cd /root/hotplug_logs/2026-04-09_topo06
/root/nvme_hotplug_scripts/4-check-Finish.sh
```

## 9. 每轮建议额外保存的环境信息

每轮开始时，在对应目录执行：

```bash
date > env_info.txt
hostname >> env_info.txt
uname -a >> env_info.txt
nvme list > nvme_list.txt
lsblk > lsblk.txt
fdisk -l > fdisk.txt
lspci -tv > lspci_tv.txt
```

保存当前盘符与拓扑映射：

```bash
for d in /dev/nvme*n1; do
    name=$(basename "$d")
    ctrl=${name%n1}
    path=$(readlink -f /sys/class/nvme/$ctrl)
    nvme_bdf=$(basename "$(dirname "$(dirname "$path")")")
    root_bdf=$(basename "$(dirname "$(dirname "$(dirname "$(dirname "$path")")")")")
    echo "$d -> NVMe:$nvme_bdf  RootPort:$root_bdf"
done | sort > nvme_topology_map.txt
```

## 10. 每次循环怎么记录

建议每次人工拔插完成后都手工追加一行结果。

示例：

```bash
echo "loop01 pull nvme0n1 PASS $(date '+%F %T')" >> loop_record.txt
echo "loop02 pull nvme1n1 PASS $(date '+%F %T')" >> loop_record.txt
```

如果异常：

```bash
echo "loop07 pull nvme0n1 FAIL md5_error $(date '+%F %T')" >> loop_record.txt
```

建议记录内容：

- 第几次
- 拔的是哪块盘
- 是否 PASS/FAIL
- 如果 FAIL，写明原因
- 时间

## 11. 每轮判定标准

每次回插后，需要关注：

- 盘是否重新识别
- `MD5` 是否一致
- `fio` 是否报错
- `dmesg` 是否有 NVMe 相关错误
- `BMC/FDM` 是否有盘相关错误

### PASS 条件

- 盘能重新识别
- `MD5` 校验通过
- `fio` 无报错
- `dmesg` 无严重盘错误
- `BMC/FDM` 无严重盘错误

### FAIL 条件示例

- 回插后盘不识别
- `MD5` 失败
- `fio` 出现 I/O error、device lost、timeout
- `dmesg` 出现 reset failed、abort、I/O error、timeout
- `BMC/FDM` 出现盘故障记录
