# NVMe 热插拔脚本执行顺序

## 1. 修改配置

先编辑 `config.sh`，只改 `DUTS`。

示例：

```bash
DUTS=(
  "/dev/nvme4n1"
  "/dev/nvme9n1"
)
```

要求：

- 不要把系统盘 `/dev/nvme5n1` 放进去。
- `DUTS` 里只放本轮需要热插拔的 2 块被测盘。

## 2. 上传脚本到服务器

把整个目录上传到 Linux 服务器，例如：

```bash
/root/nvme_hotplug_scripts
```

授权：

```bash
cd /root/nvme_hotplug_scripts
chmod +x *.sh
```

## 3. 首次使用先做分区和挂载准备

这一步每组被测盘只做一次，会清空 `DUTS` 中盘的数据。

```bash
cd /root/nvme_hotplug_scripts
./0-prepare-nvme.sh
```

这一步会做：

- 每块 DUT 建 GPT
- `p1` 划 10G
- `p2` 划剩余全部
- `p1` 格式化为 ext4
- 写入 `/etc/fstab`
- 创建挂载目录

## 4. 不带 IO 的热插拔执行顺序

每个槽位组合开始时：

```bash
./1-check-Begin.sh
./2-md5.sh
```

然后人工执行：

1. 拔盘
2. 等 30 秒
3. 慢插回去
4. 插回 6 秒后执行：

```bash
./3-check-md5.sh
```

重复：

1. 再次拔盘
2. 等 30 秒
3. 慢插回去
4. 插回 6 秒后执行 `./3-check-md5.sh`

按手册要求重复到目标次数，例如 30 次。

本轮结束后执行：

```bash
./4-check-Finish.sh
```

## 5. 带 IO 的热插拔执行顺序

每个槽位组合开始时：

```bash
./1-check-Begin.sh
./2-md5.sh
```

启动压力：

```bash
./fio.sh
```

看到 `fio-pids.log` 生成后，人工执行：

1. 拔盘
2. 等 30 秒
3. 慢插回去
4. 插回 6 秒后执行：

```bash
./3-check-md5.sh
```

如果需要继续下一轮带 IO：

```bash
./fio.sh
```

然后重复拔插和 `./3-check-md5.sh`。

达到规定次数后执行：

```bash
./4-check-Finish.sh
```

## 6. 每次检查重点

看这些日志：

- `logs/dmesg-After-Reinsert.log`
- `logs/dmesg-Finish.log`
- `logs/bmc-After-Reinsert.log`
- `logs/*-Begin.log`
- `logs/*-check-md5.log`
- `logs/*-Finish.log`
- `logs/*-fio.log`

预期：

- 盘重新识别正常
- `MD5` 一致
- `fio` 无异常报错
- `dmesg`、`BMC SEL/FDM` 无盘相关错误

## 7. 换拓扑槽位时

换到新的槽位后，修改 `config.sh` 里的 DUT 盘符，再重复：

```bash
./1-check-Begin.sh
./2-md5.sh
./fio.sh
人工拔插
./3-check-md5.sh
...
./4-check-Finish.sh
```

如果更换的是全新盘或分区已变化，先重新执行：

```bash
./0-prepare-nvme.sh
```
