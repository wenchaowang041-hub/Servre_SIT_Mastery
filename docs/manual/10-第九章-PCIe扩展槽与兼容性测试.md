# 第九章 PCIe 扩展槽与兼容性测试

## 1. 模块概述与重要性

PCIe 是现代服务器的公共底座。  
网卡、NVMe SSD、RAID 控制器、GPU、NPU、DPU、CXL 设备，几乎都要依赖 PCIe。

所以 PCIe 测试不是“一个小模块”，而是所有专项测试的基础前提。

很多高价值问题都藏在这一层：

- 卡能识别，但只跑到 x8
- 卡能识别，但代际从 Gen5 掉到 Gen4
- 某些槽位偶发掉卡
- 某个 riser 或 retimer 批次不稳定
- 多卡平台里部分设备被 BIOS 资源分配或 Above 4G 影响
- AER 一直在报 corrected error，但业务还没明显出问题

高级工程师做 PCIe 测试，不是只看“有没有设备”，而是要回答：

1. 设备是否都被正确枚举
2. 每个设备的链路能力与当前状态是否匹配
3. 槽位、riser、背板、retimer、供电是否协同正常
4. BIOS/UEFI 是否影响了设备识别或性能
5. 长时间负载下是否会出现 AER、掉卡、reset、链路重训

## 3. 基础原理讲解

### 3.1 PCIe 测试到底在测什么

至少要看五件事：

1. 设备是否识别
2. 代际是否正确
3. 宽度是否正确
4. 错误是否可接受
5. 压力下是否稳定

### 3.2 `LnkCap` 和 `LnkSta` 的区别

- `LnkCap`：设备理论支持能力
- `LnkSta`：当前实际运行状态

高级工程师最先看的不是设备名，而是这两个字段是否匹配。

### 3.3 为什么“能识别”不代表“没问题”

因为很多设备在 PCIe 层的问题不会直接表现为不可见，而是表现为：

- 吞吐低
- 时延高
- 偶发 reset
- Correctable Error 持续增长
- 某些负载下不稳定

## 4. 详细测试用例

### 用例 1：PCIe 设备全量枚举与关键设备筛选

#### 测试目的

确认所有关键 PCIe 设备均已被识别。

#### 所需工具

- `lspci`

#### 完整测试命令

```bash
lspci
lspci -nn
lspci | egrep -i "ethernet|network|raid|non-volatile|vga|3d|processing accelerators"
```

#### 重点输出字段表

| 字段 | 含义 | 正常范围 | 异常处理 |
|---|---|---|---|
| 设备行 | 设备识别记录 | 与实配一致 | 漏设备：查供电/插槽/BIOS |
| `Vendor:Device ID` | 设备 ID | 可匹配厂商型号 | 不匹配：查驱动和兼容性 |

### 用例 2：链路代际与宽度检查

#### 测试目的

检查关键设备是否以目标代际和宽度运行。

#### 完整测试命令

```bash
lspci -vv
```

#### 重点输出字段表

| 字段 | 含义 | 正常范围 | 异常处理 |
|---|---|---|---|
| `LnkCap` | 支持的最大能力 | 由设备决定 | 仅用于对比 |
| `LnkSta` | 当前运行能力 | 应接近 `LnkCap` | 降速/降宽：查 BIOS、槽位、riser |
| `ASPM` | 电源管理策略 | 按平台策略配置 | 异常影响性能/稳定性 |

### 用例 3：AER 与内核错误检查

#### 测试目的

检查是否存在 Correctable/Uncorrectable PCIe 错误。

#### 完整测试命令

```bash
dmesg -T | egrep -i "pcie|aer|corrected|uncorrected|fatal"
journalctl -k | egrep -i "pcie|aer|corrected|uncorrected|fatal"
```

### 用例 4：换槽与兼容性验证

#### 测试步骤

1. 记录设备在原槽位的 `LnkSta`。
2. 更换到目标槽位。
3. 重新启动并记录 `LnkSta`。
4. 对比两次结果，判断槽位、riser 或背板是否有差异。

## 5. 结果分析与问题诊断方法

排查顺序建议：

1. 先确认设备是否识别
2. 再看 `LnkCap/LnkSta`
3. 再看 BIOS PCIe 设置
4. 再看 riser/backplane/retimer
5. 最后看供电和热

## 6. 最佳实践与注意事项

1. 多卡平台必须做换槽验证。
2. GPU/NPU/NIC/SSD 的 PCIe 问题常常不是单卡问题，而是槽位或背板问题。
3. AER 持续增长即使暂时不影响业务，也要记录为风险。
